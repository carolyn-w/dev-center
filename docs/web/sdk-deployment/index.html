Deployment

Deploying your library using Geocortex Web Designer#

The easiest way to deploy your custom library built using the SDK is to upload it to an application using the Geocortex Web Designer.

warning

This is a temporary deployment pattern for the preview release. We're actively improving this pattern, and as such it will change during the preview phase.

Modifying Layout and App Config using the Geocortex Web Designer#

Geocortex Web Designer provides a powerful interface for editing the layout and configuration of built-in components, but if you need to add a custom component to your application, you will need to upload custom layout and app JSON.

  1. Type Ctrl + Shift + E while focused in the Geocortex Web Designer to open up the experimental development editing interface.
  2. Copy and paste the app.json and layout.xml from your local SDK project to the Geocortex Web Designer Interface.
note

You may need to add an app item for the referenced layout file to the app config in the Designer Interface.

{
"schemaVersion": "1.0",
"items": [
...
{
"$type":"layout",
"id":"desktop-landscape",
"title":"Desktop Landscape",
"url":"./desktop-landscape.xml"
}
...
]
}

Uploading Custom Code to an App#

  1. Run npm run build in the root of your project to create a production build of your library
    • This creates an optimized production build that is output to the build folder of your project. This build is designed for use in production
  2. Open the application in Geocortex Web Designer (on-premises or SaaS, depending on your production environment) that you want to upload your custom code to
  3. Type Ctrl + Shift + E while focused in the Geocortex Web Designer to open up the experimental development editing interface
    • This interface is for debugging and development purposes only, and is not intended for general configuration of your application.
  4. Upload the production build of your custom library located in the build folder to the Custom Code section of the development editing interface
  5. Copy and paste the custom layout and app config.

Deploy the app#

After you've uploaded your custom library to your application, you can push the application through the configured deployment stages using the deployment pipeline in the Designer. Your library code will be copied through the deployment stages, so you can be confident that changes to your library code in the development stage won't affect your production app until you explicitly deploy to the production stage.
