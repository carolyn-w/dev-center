Customize the Look and Feel

Geocortex Web's appearance is built around two concepts: theme and layout. The theme controls the color scheme of the app, while the layout controls the organization of content.

#Prerequisites

  • Follow along by setting up the Geocortex Web SDK and editing the minimal layout and app config provided.
  • Check out the deployment instructions to learn about how to deploy layout and app config to an application.

#Seeding a New Theme from an Existing Theme

Two predefined themes are included with Geocortex Web, light and dark. You can select a predefined theme and accent color with Geocortex Web Designer.

The branding service is responsible for the theming of the application. To select a theme and accent color from within the config, you can configure the branding service in the app config. The template property is used to specify the base theme to seed colors from.

  • App Config
  • UI
{
"schemaVersion": "1.0",
"items": [
...
{
"$type": "branding",
"id": "branding",
"activeTheme": "custom-vampire",
"themes": [{
"id": "custom-vampire",
"accentColor": "red",
"template": "dark"
}]
},
...
]
}

#Customizing Theme Colors

Every color in a theme can be fully customized by editing the colors property of the the branding service configuration.

tip

Colors can be specified with their common name, a hex value, or a rgba value.

  • App Config
  • UI
{
"schemaVersion": "1.0",
"items": [
...
{
"$type": "branding",
"id": "branding",
"activeTheme": "custom-vampire",
"themes": [{
"id": "custom-vampire",
"accentColor": "red",
"template": "dark",
"colors": {
"emphasizedButtonBackground": "purple",
"primaryAccentLarge": "#00b7eb",
"secondaryBackground": [249, 77, 0, 255]
}
}]
},
...
]
}

#Using Theme Colors In Custom Components

When creating custom components, you may need to style your component. If you use Geocortex UI Elements, these elements already participate in the theme, so your element will be themed automatically. However, you can also use the various theming colors in custom css, through the var(...) css function.

.my-element {
background-color: var(--primaryBackground);
}
